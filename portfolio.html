Engineering Decisions

Why things are built the way they are.

The choices that separate production systems from demo projects.

Money Storage

Integers, not floats

float64 arithmetic introduces precision errors that compound silently at volume. All monetary values are stored as int64 in the smallest currency unit — always exact.

Accounting Model

Double-entry, always

Every financial movement creates two ledger entries. No balance column is mutated directly. The ledger is append-only and immutable — the same model real banks use.

API Safety

Idempotency by default

Mutation endpoints require an Idempotency-Key. Network retries cannot cause duplicate operations. The server stores and replays results without re-executing.

AI Retrieval

RAG over naive prompting

LLMs hallucinate without grounding. Retrieval-augmented generation pulls real context from a knowledge base before generation — results are grounded in actual data.

Search

Embeddings over keywords

Keyword matching misses synonyms, context, and intent. Embedding-based similarity search finds meaning — a search for "payments engineer" surfaces "fintech backend developer."

Testing

Real DB, not mocks

Integration tests hit a real PostgreSQL instance with actual SQL constraints and ACID semantics. Mocks prove your mock works — real tests prove your system works.
